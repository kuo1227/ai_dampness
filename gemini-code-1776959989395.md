# AI 中醫濕氣檢測小幫手 - 產品需求文件 (PRD)

## 1. 專案概觀
- **專案名稱**：AI 中醫濕氣檢測小幫手
- **開發者/團隊**：個人開發者
- **目標平台**：LINE 官方帳號 (LINE OA)
- **技術棧**：Cloudflare Workers, Cloudflare D1 (Database), NVIDIA NIM (Gemma 3 27B-it API)

## 2. 產品定位與目標
本產品旨在透過 AI 模擬中醫師的問診流程，結合 LINE 的動態選單與結構化訊息（Quick Replies, Flex Messages），提供使用者一個直觀、專業且低門檻的「體內濕氣」自我檢測工具。

## 3. 系統架構
- **後端控制中心**：Cloudflare Workers (負責處理 LINE Webhook、狀態控制與 API 調度)。
- **資料持久化**：Cloudflare D1 (記錄用戶狀態 `current_step`、問診記錄與檢測結果)。
- **AI 腦核心**：NVIDIA NIM 上的 Gemma 3 27B-it (負責症狀分析與健康建議生成)。
- **通訊介面**：LINE Messaging API。

## 4. 功能需求 (Functional Requirements)

### F1: 引導式問診流 (Workflow)
1. **觸發**：用戶點擊 Rich Menu「開始檢測」。
2. **多輪問答**：系統根據 D1 記錄的步進值 (Step 1-6)，發送特定問題（如：食慾、大便型態、體感重量、舌苔等）。
3. **輸入優化**：每個問題皆附帶 3-4 個 `quickReply` 按鈕，避免用戶手動輸入文字。

### F2: AI 診斷分析
- 在完成所有問題採集後，將數據封裝傳送至 Gemma 3。
- **Prompt 要求**：輸出需包含「濕氣類型（如痰濕、濕熱）」、「嚴重程度(%)」、「食療建議」與「穴位按摩」。

### F3: 診斷報告輸出 (Flex Message)
- 使用 LINE Flex Message 提供視覺化報告，包含色塊標示（紅：嚴重、綠：健康）。
- 附帶「分享給好友」或「預約專業諮詢」按鈕。
